## <a name="basic-raw-sql-queries"></a><span data-ttu-id="4e602-107">Consultas SQL básicas sin formato</span><span class="sxs-lookup"><span data-stu-id="4e602-107">Basic raw SQL queries</span></span>


## <a name="passing-parameters"></a><span data-ttu-id="4e602-110">Pasar parámetros</span><span class="sxs-lookup"><span data-stu-id="4e602-110">Passing parameters</span></span>

<span data-ttu-id="4e602-111">Al igual que con cualquier API que acepta código SQL, es importante parametrizar cualquier entrada de usuario como protección contra un ataque por inyección de código SQL.</span><span class="sxs-lookup"><span data-stu-id="4e602-111">As with any API that accepts SQL, it is important to parameterize any user input to protect against a SQL injection attack.</span></span> <span data-ttu-id="4e602-112">Puede incluir marcadores de posición de parámetro en la cadena de la consulta y luego suministrar valores de parámetro como argumentos adicionales.</span><span class="sxs-lookup"><span data-stu-id="4e602-112">You can include parameter placeholders in the SQL query string and then supply parameter values as additional arguments.</span></span> <span data-ttu-id="4e602-113">Cualquier valor de parámetro que suministre se convertirá automáticamente en `DbParameter`.</span><span class="sxs-lookup"><span data-stu-id="4e602-113">Any parameter values you supply will automatically be converted to a `DbParameter`.</span></span>

<span data-ttu-id="4e602-114">En el ejemplo siguiente, se pasa un parámetro único a un procedimiento almacenado.</span><span class="sxs-lookup"><span data-stu-id="4e602-114">The following example passes a single parameter to a stored procedure.</span></span> <span data-ttu-id="4e602-115">Si bien esto se puede ver como la sintaxis `String.Format`, el valor suministrado se encapsula en un parámetro y el nombre del parámetro generado se inserta donde se especificó el marcador de posición `{0}`.</span><span class="sxs-lookup"><span data-stu-id="4e602-115">While this may look like `String.Format` syntax, the supplied value is wrapped in a parameter and the generated parameter name inserted where the `{0}` placeholder was specified.</span></span>

``` csharp

    .ToList();
```

<span data-ttu-id="4e602-116">Se trata de la misma consulta, pero usando la sintaxis de interpolación de cadena, que se admite en EF Core 2.0 y versiones superiores:</span><span class="sxs-lookup"><span data-stu-id="4e602-116">This is the same query but using string interpolation syntax, which is supported in EF Core 2.0 and above:</span></span>

<!-- [!code-csharp[Main](samples/core/Querying/RawSQL/Sample.cs)] -->
``` csharp
var user = "johndoe";

var blogs = context.Blogs
    .FromSql($"EXECUTE dbo.GetMostPopularBlogsForUser {user}")
    .ToList();
```

<span data-ttu-id="4e602-117">También puede construir un elemento DbParameter y suministrarlo como un valor de parámetro:</span><span class="sxs-lookup"><span data-stu-id="4e602-117">You can also construct a DbParameter and supply it as a parameter value:</span></span>

``` csharp

    .FromSql("EXECUTE dbo.GetMostPopularBlogsForUser @user", user)
    .ToList();
```

<span data-ttu-id="4e602-118">Esto permite usar los parámetros con nombre en la cadena de consulta SQL, lo que resulta útil cuando un procedimiento almacenado tiene parámetros opcionales:</span><span class="sxs-lookup"><span data-stu-id="4e602-118">This allows you to use named parameters in the SQL query string, which is useful when a stored procedure has optional parameters:</span></span>

``` csharp

    .FromSql("EXECUTE dbo.GetMostPopularBlogs @filterByUser=@user", user)
    .ToList();
```

## <a name="composing-with-linq"></a><span data-ttu-id="4e602-119">Redacción con LINQ</span><span class="sxs-lookup"><span data-stu-id="4e602-119">Composing with LINQ</span></span>

<span data-ttu-id="4e602-120">Si la consulta SQL admite redacción en la base de datos, puede redactarla sobre la consulta SQL sin formato inicial mediante operadores LINQ.</span><span class="sxs-lookup"><span data-stu-id="4e602-120">If the SQL query can be composed on in the database, then you can compose on top of the initial raw SQL query using LINQ operators.</span></span> <span data-ttu-id="4e602-121">Las consultas SQL que se pueden redactar empiezan con la palabra clave `SELECT`.</span><span class="sxs-lookup"><span data-stu-id="4e602-121">SQL queries that can be composed on begin with the `SELECT` keyword.</span></span>

<span data-ttu-id="4e602-122">En el ejemplo siguiente, se usa una consulta SQL sin formato que se selecciona de una función con valores de tabla (TVF) y luego se redacta en ella con LINQ para realizar filtrado y ordenación.</span><span class="sxs-lookup"><span data-stu-id="4e602-122">The following example uses a raw SQL query that selects from a Table-Valued Function (TVF) and then composes on it using LINQ to perform filtering and sorting.</span></span>

<!-- [!code-csharp[Main](samples/core/Querying/RawSQL/Sample.cs)] -->
``` csharp
var searchTerm = ".NET";

var blogs = context.Blogs
    .FromSql($"SELECT * FROM dbo.SearchBlogs({searchTerm})")
    .Where(b => b.Rating > 3)
    .OrderByDescending(b => b.Rating)
    .ToList();
```

## <a name="change-tracking"></a><span data-ttu-id="4e602-123">Seguimiento de cambios</span><span class="sxs-lookup"><span data-stu-id="4e602-123">Change Tracking</span></span>

<span data-ttu-id="4e602-124">Las consultas que usan `FromSql()` siguen las mismas reglas de seguimiento de cambios que las demás consultas LINQ en EF Core.</span><span class="sxs-lookup"><span data-stu-id="4e602-124">Queries that use the `FromSql()` follow the exact same change tracking rules as any other LINQ query in EF Core.</span></span> <span data-ttu-id="4e602-125">Por ejemplo, si la consulta proyecta tipos de entidad, se realizará un seguimiento de los resultados de forma predeterminada.</span><span class="sxs-lookup"><span data-stu-id="4e602-125">For example, if the query projects entity types, the results will be tracked by default.</span></span>  

<span data-ttu-id="4e602-126">En el ejemplo siguiente se usa una consulta SQL sin formato que realiza una selección en una función con valores de tabla (TVF) y, a continuación, deshabilita el seguimiento de cambios con la llamada a .AsNoTracking():</span><span class="sxs-lookup"><span data-stu-id="4e602-126">The following example uses a raw SQL query that selects from a Table-Valued Function (TVF), then disables change tracking with the call to .AsNoTracking():</span></span>

<!-- [!code-csharp[Main](samples/core/Querying/RawSQL/Sample.cs)] -->
``` csharp
var searchTerm = ".NET";

var blogs = context.Query<SearchBlogsDto>()
    .FromSql($"SELECT * FROM dbo.SearchBlogs({searchTerm})")
    .AsNoTracking()
    .ToList();
```

## <a name="including-related-data"></a><span data-ttu-id="4e602-127">Inclusión de datos relacionados</span><span class="sxs-lookup"><span data-stu-id="4e602-127">Including related data</span></span>

<span data-ttu-id="4e602-128">El método `Include()` puede usarse para incluir datos relacionados, igual que cualquier otra consulta LINQ:</span><span class="sxs-lookup"><span data-stu-id="4e602-128">The `Include()` method can be used to include related data, just like with any other LINQ query:</span></span>

``` csharp
var searchTerm = ".NET";

var blogs = context.Blogs
    .FromSql($"SELECT * FROM dbo.SearchBlogs({searchTerm})")
    .Include(b => b.Posts)
    .ToList();
```

## <a name="limitations"></a><span data-ttu-id="4e602-129">Limitaciones</span><span class="sxs-lookup"><span data-stu-id="4e602-129">Limitations</span></span>

<span data-ttu-id="4e602-130">Existen algunas limitaciones que debe considerar al usar las consultas SQL sin formato:</span><span class="sxs-lookup"><span data-stu-id="4e602-130">There are a few limitations to be aware of when using raw SQL queries:</span></span>

* <span data-ttu-id="4e602-131">La consulta SQL debe devolver datos para todas las propiedades del tipo de entidad o de consulta.</span><span class="sxs-lookup"><span data-stu-id="4e602-131">The SQL query must return data for all properties of the entity or query type.</span></span>

* <span data-ttu-id="4e602-132">Los nombres de las columnas del conjunto de resultados deben coincidir con los nombres de las columnas a los que se asignan las propiedades.</span><span class="sxs-lookup"><span data-stu-id="4e602-132">The column names in the result set must match the column names that properties are mapped to.</span></span> <span data-ttu-id="4e602-133">Observe que esto es diferente de lo que ocurre en EF6, donde las consultas SQL omitían la asignación de propiedades y columnas, y los nombres de las columnas del conjunto de resultados tenían que coincidir con los nombres de las propiedades.</span><span class="sxs-lookup"><span data-stu-id="4e602-133">Note this is different from EF6 where property/column mapping was ignored for raw SQL queries and result set column names had to match the property names.</span></span>

* <span data-ttu-id="4e602-134">La consulta SQL no puede incluir datos relacionados.</span><span class="sxs-lookup"><span data-stu-id="4e602-134">The SQL query cannot contain related data.</span></span> <span data-ttu-id="4e602-135">Sin embargo, en muchos casos puede redactar sobre la consulta si usa el operador `Include` para devolver datos relacionados (consulte [Inclusión de datos relacionados](#including-related-data)).</span><span class="sxs-lookup"><span data-stu-id="4e602-135">However, in many cases you can compose on top of the query using the `Include` operator to return related data (see [Including related data](#including-related-data)).</span></span>

* <span data-ttu-id="4e602-136">Las instrucciones `SELECT` que se pasan a este método por lo general admiten composición: Si EF Core debe evaluar operadores de consulta adicionales en el servidor (por ejemplo, para trasladar los operadores LINQ aplicados después de `FromSql`), el código SQL suministrado se considerará una subconsulta.</span><span class="sxs-lookup"><span data-stu-id="4e602-136">`SELECT` statements passed to this method should generally be composable: If EF Core needs to evaluate additional query operators on the server (for example, to translate LINQ operators applied after `FromSql`), the supplied SQL will be treated as a subquery.</span></span> <span data-ttu-id="4e602-137">Esto significa que el código SQL que se pasa no contendrá ningún carácter ni opción que no sea válido en una subconsulta, como:</span><span class="sxs-lookup"><span data-stu-id="4e602-137">This means that the SQL passed should not contain any characters or options that are not valid on a subquery, such as:</span></span>
  * <span data-ttu-id="4e602-138">un punto y coma final</span><span class="sxs-lookup"><span data-stu-id="4e602-138">a trailing semicolon</span></span>
  * <span data-ttu-id="4e602-139">En SQL Server, una sugerencia en el nivel de consulta final (por ejemplo, `OPTION (HASH JOIN)`)</span><span class="sxs-lookup"><span data-stu-id="4e602-139">On SQL Server, a trailing query-level hint (for example, `OPTION (HASH JOIN)`)</span></span>
  * <span data-ttu-id="4e602-140">En SQL Server, cláusula `ORDER BY` que no va acompañada por `OFFSET 0` O BIEN `TOP 100 PERCENT` en la cláusula `SELECT`</span><span class="sxs-lookup"><span data-stu-id="4e602-140">On SQL Server, an `ORDER BY` clause that is not accompanied of `OFFSET 0` OR `TOP 100 PERCENT` in the `SELECT` clause</span></span>
